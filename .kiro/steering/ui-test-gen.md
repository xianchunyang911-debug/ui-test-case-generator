# UI走查用例生成规则

当用户请求生成UI走查用例时，请遵循以下规则：

## 触发条件
- 用户明确请求："生成UI走查用例"
- 用户保存包含"需求文档"的Markdown文件
- 用户在Chat中使用 #ui-test-gen 上下文

## 自动识别输出格式
根据需求文档的复杂度自动选择格式：
- **Excel多Sheet格式**：用例数 > 50 或 模块数 > 3（推荐）
- **CSV格式**：用例数 ≤ 50 且 模块数 ≤ 3

## 生成流程

### 1. 分析需求文档
- 读取需求文档内容
- 识别功能模块（根据一级、二级标题）
- 提取页面结构和字段说明
- 识别上下游模块关系

### 2. 生成UI走查用例
应用8大UI走查原则：
1. 视觉一致性原则
2. 布局与响应式原则
3. 组件状态完整性原则
4. 内容与文案准确性原则
5. 交互与反馈原则
6. 可访问性与可用性原则
7. 场景法
8. 异常与负向流程验证

### 3. 生成输出文件
- UI走查用例文件（CSV或Excel）
- UI走查计划文档（Markdown）
- 快速开始指南（Markdown）

## 输出目录
- 用例文件：`UI用例/`
- 计划文档：项目根目录
- 使用说明：项目根目录

## 质量标准
- 用例覆盖率 > 90%
- 高优先级用例占比 > 60%
- 每个模块至少5个用例
- 包含场景流程和异常场景

## 注意事项
1. 保持用例编号连续（UI-TC001, UI-TC002...）
2. 优先级合理分配（高/中/低）
3. 预期结果具体可验证
4. 检查项描述清晰明确
